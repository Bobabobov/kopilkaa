# Совместимость с браузерами

Проект должен работать на старых телефонах и браузерах (в т.ч. Android 5+, in-app браузеры Telegram и т.п.). Иначе у части пользователей будут ошибки вроде «Что-то пошло не так».

## Как не допускать поломок

### 1. Не полагаться на свежий JavaScript

- Методы вроде `Array.prototype.findLast`, `findLastIndex`, `at()` появились в ES2022/ES2023 — в старых средах их нет.
- В коде лучше использовать то, что поддерживается везде: цикл, `arr[arr.length - 1]`, `find` + `reverse()` и т.п., либо добавлять полифилл (как для `findLast` в `app/layout.tsx`).

### 2. Проверять поддержку перед использованием

Перед использованием «нового» API можно проверить поддержку и при необходимости подставить полифилл или альтернативу:

```ts
if (typeof Array.prototype.findLast !== "function") {
  // полифилл или другая реализация
}
```

### 3. Browserslist

В `package.json` задан `browserslist` — под него работают Autoprefixer и часть инструментов. При добавлении зависимостей учитывай, что их код тоже может использовать современные API; при необходимости добавляй полифиллы в корневой layout.

### 4. Проверка на реальных устройствах

- Периодически открывать ключевые страницы (главная, заявки, отзывы, профиль) на старом Android или в режиме эмуляции «медленного»/старого устройства в Chrome DevTools (Throttling, устройство из списка).
- Либо использовать [BrowserStack](https://www.browserstack.com/) / аналог и проверить несколько старых версий Android и iOS.

### 5. Понятные ошибки для пользователя

На страницах с формами и критичным потоком (например, `/applications`) уже есть error boundary с кнопкой «Показать подробности». Пользователь может прислать скриншот текста ошибки — по нему проще искать причину (в т.ч. «X is not a function» из-за отсутствующего API).

### 6. Что уже сделано

- В `app/layout.tsx` подключён полифилл для `Array.prototype.findLast` (скрипт с `beforeInteractive`), чтобы старые браузеры не падали на коде из зависимостей.
- В `hooks/applications/formState/useTrustAndReview.ts` добавлена защита от невалидных данных с API (уровень доверия, лимиты), чтобы не было падений из-за неожиданного формата ответа.
